## 二、数据库表功能与存储内容详解

### 2.1 用户与权限

**role**  
- 功能：系统角色定义  
- 存储：id、code(buyer/merchant/admin)、name  
- 初始数据：买家、商家、管理员  

**user**  
- 功能：用户账户  
- 存储：用户名、密码、昵称、手机、邮箱、头像、状态(0禁用/1正常)  

**user_role**  
- 功能：用户与角色多对多关联  
- 存储：user_id、role_id  

**permission**（未使用）  
- 功能：预留权限码  
- 存储：权限编码、名称  

### 2.2 商品相关

**category**  
- 功能：商品分类（如 CPU、显卡、主板）  
- 存储：parent_id、name、sort  

**brand**  
- 功能：品牌（如 Intel、AMD）  
- 存储：name、logo、sort  

**product**  
- 功能：商品主表  
- 存储：merchant_id、category_id、brand_id、name、subtitle、main_image、images、detail、price、stock、status(0待审/1上架/2下架)、audit_status(0待审/1通过/2拒绝)、audit_remark  

**product_spec**（未使用）  
- 功能：预留商品规格  
- 存储：product_id、spec_name、spec_value  

### 2.3 购物与订单

**cart_item**  
- 功能：购物车项  
- 存储：user_id、product_id、quantity  
- 约束：同一用户同一商品唯一  

**order**  
- 功能：订单主表  
- 存储：order_no、user_id、merchant_id、total_amount、pay_amount、discount_amount、status(PENDING_PAY/PAID/PENDING_SHIP/SHIPPED/COMPLETED/CANCELLED)、pay_channel、pay_trade_no、pay_status、pay_time、receiver_name/phone/address、remark、rating(1-5星)  

**order_item**  
- 功能：订单明细  
- 存储：order_id、product_id、product_name、product_image、price、quantity、total  

**order_log**  
- 功能：订单状态变更记录  
- 存储：order_id、status、remark  

**logistics**  
- 功能：物流信息  
- 存储：order_id、company、tracking_no、status  

### 2.4 促销

**promotion**  
- 功能：促销活动  
- 存储：merchant_id、name、type(FULL_REDUCE/PERCENT)、start_time、end_time、status  

**promotion_rule**  
- 功能：促销规则  
- 存储：promotion_id、rule_key(full_amount/reduce_amount/discount_percent)、rule_value  

### 2.5 DIY 装机

**diy_template**  
- 功能：DIY 配件类型  
- 存储：name(CPU/主板/内存/硬盘/显卡/电源/机箱)、sort  

**diy_compatibility_rule**  
- 功能：商品与 DIY 类型关联，决定某商品可作为哪种配件选择  
- 存储：template_id、product_id、compatible_with(JSON，预留)  

**diy_incompatible_pair**  
- 功能：不兼容配件对，用于兼容性检测  
- 存储：product_id_a、product_id_b、remark  

### 2.6 资讯与社区

**article**  
- 功能：资讯文章  
- 存储：title、summary、content、cover、author_id、status  

**comment**  
- 功能：资讯评论  
- 存储：article_id、user_id、parent_id、content、status  

### 2.7 客服与消息

**message**  
- 功能：站内消息/客服对话  
- 存储：from_user_id、to_user_id、content、is_read  

### 2.8 系统功能

**sensitive_word**  
- 功能：敏感词库  
- 存储：word  

**ad_position**  
- 功能：广告位定义  
- 存储：code、name  

**ad**  
- 功能：广告内容  
- 存储：position_id、title、image、link、sort、status、start_time、end_time  

**system_config**  
- 功能：系统配置键值对  
- 存储：config_key、config_value  

**audit_log**（未使用）  
- 功能：预留审计日志  
- 存储：user_id、module、action、detail

## 三、项目整体系统结构

### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端 (Vue 3 + Vite)                      │
│  Element Plus | Vue Router | Pinia | Axios | ECharts        │
└─────────────────────────┬───────────────────────────────────┘
                          │ HTTP/REST API
┌─────────────────────────▼───────────────────────────────────┐
│                   后端 (Spring Boot 3.2)                     │
│  Controller → Service → Mapper → MySQL                      │
│  JWT 认证 | MyBatis | 统一 Result 封装                       │
└─────────────────────────┬───────────────────────────────────┘
                          │ JDBC
┌─────────────────────────▼───────────────────────────────────┐
│                    MySQL 数据库 (zhiwei)                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 目录结构

**前端 (frontend/)**  
- `src/views/buyer/` - 买家端页面（首页、商品、购物车、订单、DIY、天梯图、系统下载、资讯、智能客服）  
- `src/views/merchant/` - 商家端（商品管理、订单、促销、统计）  
- `src/views/admin/` - 管理后台（用户、商品审核、订单、分类、品牌、资讯、统计）  
- `src/stores/` - Pinia 状态（用户）  
- `src/utils/` - 请求封装、工具  

**后端 (backend/)**  
- `controller/` - REST 接口  
- `service/` - 业务逻辑  
- `mapper/` - MyBatis 数据访问  
- `entity/` - 实体类  
- `config/` - 配置（CORS、拦截器）  
- `interceptor/` - JWT 认证拦截  
- `common/` - Result 封装  

### 3.3 角色与模块

| 角色 | 主要功能 |
|------|----------|
| 买家(buyer) | 浏览商品、购物车、下单、DIY装机、资讯、智能客服、订单管理、满意度评价 |
| 商家(merchant) | 商品管理、订单处理、促销管理、数据统计 |
| 管理员(admin) | 用户管理、商品审核、订单管理、分类/品牌/资讯、数据统计 |

## 五、系统架构设计

### 5.1 前后端分离

- 前端：Vue 3 + Vite，独立部署，通过 Axios 调用后端 API  
- 后端：Spring Boot REST API，统一返回 `{ code, msg, data }`  
- 认证：JWT，放在请求头 `Authorization: Bearer <token>`  

### 5.2 三层架构

- Controller：接收请求、参数校验、调用 Service、返回结果  
- Service：业务逻辑、事务控制  
- Mapper：MyBatis 数据访问  

### 5.3 权限控制

- 角色：buyer、merchant、admin  
- 认证：AuthInterceptor 校验 JWT，解析 userId、role  
- 放行：登录、注册、商品列表/详情、分类、品牌、文章、评论、广告、DIY、智能客服等  
- 路由：前端根据 role 控制页面访问

## 八、Windows 新环境部署指南

### 8.1 环境准备

#### 8.1.1 所需软件

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| JDK | 17+ | 后端运行 |
| Maven | 3.6+ | 后端构建 |
| Node.js | 18+ | 前端构建与开发 |
| MySQL | 8.0+ | 数据库 |
| Nginx | 1.20+ | 生产环境反向代理（可选） |

#### 8.1.2 安装步骤

1. **JDK 17**
   - 下载：https://adoptium.net/ 或 Oracle JDK
   - 安装后配置 `JAVA_HOME` 环境变量
   - 验证：`java -version`

2. **Maven**
   - 下载：https://maven.apache.org/download.cgi
   - 解压到如 `C:\Program Files\apache-maven-3.9.x`
   - 配置 `MAVEN_HOME`，并将 `%MAVEN_HOME%\bin` 加入 Path
   - 验证：`mvn -v`

3. **Node.js**
   - 下载：https://nodejs.org/（建议 LTS）
   - 安装后自动配置 npm
   - 验证：`node -v` 与 `npm -v`

4. **MySQL 8.0**
   - 下载：https://dev.mysql.com/downloads/installer/
   - 安装时设置 root 密码，端口默认 3306
   - 启动 MySQL 服务

### 8.2 数据库配置与初始化

#### 8.2.1 创建数据库用户

在 MySQL 命令行或图形工具中执行：

```sql
CREATE USER 'zhiwei'@'localhost' IDENTIFIED BY 'zhiwei';
CREATE DATABASE IF NOT EXISTS zhiwei DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
GRANT ALL PRIVILEGES ON zhiwei.* TO 'zhiwei'@'localhost';
FLUSH PRIVILEGES;
```

可按需修改用户名和密码，并同步修改后端配置。

#### 8.2.2 执行建表脚本

1. 进入项目 `database` 目录
2. 按顺序执行：
   - `schema.sql`（建表）
   - `migrate-main_image-mediumtext.sql`（主图字段类型）
   - `migrate-order-rating.sql`（订单满意度）
   - `migrate-diy-incompatible.sql`（DIY 不兼容表）
   - `init-data.sql`（初始管理员 admin/123456、DIY 模板、广告位等）

示例（命令行）：


### 8.6 前后端分离部署配置

#### 8.6.1 开发环境（本地）

- 前端：`npm run dev`，端口 5173
- 后端：`mvn spring-boot:run`，端口 8080
- 前端通过 Vite 代理访问后端：
  ```javascript
  // vite.config.ts
  server: { proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true } } }
  ```
- 前端请求 `baseURL: '/api'`，被代理到 `http://localhost:8080/api`

#### 8.6.2 生产环境 - 方案一：Nginx 反向代理

前端与后端部署在同一台服务器，由 Nginx 统一转发：

```
用户 → http://your-domain.com
        ├── /         → 前端静态文件（dist/）
        └── /api      → http://127.0.0.1:8080
```

**Nginx 示例配置**（`nginx.conf` 或站点配置）：

```nginx
server {
    listen 80;
    server_name your-domain.com;   # 或 localhost、IP

    # 前端静态文件
    location / {
        root C:/path/to/zhiwei/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

修改 `root` 为实际 `dist` 路径。

#### 8.6.3 生产环境 - 方案二：前后端分域名部署

- 前端：`https://www.example.com`（静态托管或 Nginx）
- 后端：`https://api.example.com`（8080 或 80）

需修改前端 API 地址。新建 `frontend/.env.production`：

```
VITE_API_BASE=/api
```

若后端独立域名，例如 `https://api.example.com`，则：

```
VITE_API_BASE=https://api.example.com
```

修改 `src/utils/request.ts`：

```javascript
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE || '/api',
  timeout: 10000
})
```

重新打包：`npm run build`

#### 8.6.4 后端 CORS 配置

前后端不同源时，后端需允许跨域。当前 `WebConfig` 已配置：

```java
registry.addMapping("/api/**").allowedOrigins("*").allowedMethods("*").allowedHeaders("*");
```

生产环境建议将 `*` 改为具体前端域名。

### 8.8 常见问题

| 问题 | 排查方向 |
|------|----------|
| 数据库连接失败 | 检查 MySQL 服务、URL、用户名密码、防火墙 |
| Hikari 连接校验失败 | 确认 `max-lifetime`、`connection-timeout` 配置 |
| 前端 401 | 检查 token、登录状态、后端拦截放行路径 |
| 前端请求 404 | 检查 baseURL、Nginx 代理、后端端口 |
| 静态资源 404（生产） | 检查 `try_files` 与 dist 路径 |
